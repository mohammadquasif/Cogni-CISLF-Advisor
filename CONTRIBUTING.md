# Contributing to Cogni CISLF Advisor 🚀

First off, thank you for considering contributing to the Cogni CISLF Advisor! It's people like you who make this a helpful resource for technology leaders and researchers.

## How Can I Contribute?

### Reporting Bugs
- Search the existing issues to ensure the bug hasn't been reported.
- If it's a new issue, open a bug report template and fill out the details including step-by-step reproduction.

### Suggesting Enhancements
- Open a feature request issue.
- Describe the feature, why it is useful, and potential implementation approaches.

### Pull Requests
1. Fork the repository.
2. Create your feature branch (`git checkout -b feature/AmazingFeature`).
3. Commit your changes (`git commit -m 'Add some AmazingFeature'`).
4. Push to the branch (`git push origin feature/AmazingFeature`).
5. Open a Pull Request.

## Coding Standards
- Maintain modular architecture across `app.py`, `cislf_engine.py`, `llm_providers.py`, and `manual_engine.py`.
- Keep credential vaults local and fully encrypted (`local_storage.py`).
- Follow Python PEP-8 coding style guidelines.

## Academic Integrity
The core Comprehensive Intelligent Strategic Leadership Framework (CISLF) and its underlying academic models are based on the doctoral research of Mohammad Quasif. When contributing, please ensure any additions align with the strategic leadership, socio-technical, and responsible AI governance pillars of the thesis.

Thank you for your contributions!
